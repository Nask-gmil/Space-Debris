### 症状・要望

    背景を暗くする `#drawerBackdrop` を追加。
- `src/style.css`
  - スマホ幅(820px以下)では、入力パネル・情報パネルを
    `position: fixed` の重ね表示に変更。これによりパネルの開閉が
    canvas(宇宙空間)のレイアウトに一切影響しなくなり、
    「詳細を見ると宇宙が横長に潰れる」問題を解消。
  - 普段は `panel-body` の `max-height: 0` で中身を隠し、
    `.is-expanded` クラスが付くと `max-height` を画面の高さいっぱいまで広げ、
    かつ `overflow-y: auto` でスクロール可能にした(削除ボタンまで届くように)。
  - 入力パネルは上から、情報パネルは下から広がるようにした

  - 食べ物の登録が完了したら、入力ドロワーを自動的に閉じて
    宇宙の様子がすぐ見えるようにした。
  - デスクトップ表示ではハンドルがCSSで非表示になるため、
    これらの開閉ロジックは一切見た目に影響しない(常に展開表示のまま)。

## 【修正済み】スマホ表示の崩れ(旧・3列グリッドの崩れ)


### 症状
GitHub Pagesにアップして実機(スマホ)で開くと、食べ物登録フォームの入力欄が
横並びの3列グリッドに無理やり詰め込まれ、文字が見切れて操作できない状態になっていた。

### 原因
`style.css` の `@media (max-width: 820px)` 内で、フォーム全体
(ジャンル選択・ジャンル入力・食べ物名・カロリー・登録ボタンの5要素)を
`grid-template-columns: 1fr 1fr auto` という3列のグリッドに押し込めていた。
5要素に対して3列は足りず、各セルの幅が極端に狭くなっていた。

### 修正内容
- `src/style.css`
  - スマホ幅では1カラムに縦積みするレイアウトに変更(`display:flex; flex-direction:column`)。
  - 各入力欄のラベルを非表示にせず表示するようにした(何を入力する欄か分かるように)。
  - 入力欄の高さを48pxに拡大し、指でタップしやすくした。
  - フォントサイズを16pxにした(15px以下だとiOS Safariがフォーカス時に自動ズームしてしまうため)。
  - `.app` と `body` の高さ指定を `auto` にして、スマホで内容が縦に伸びてもレイアウトが壊れないようにした。

### 動作確認のお願い
`npm run dev` した状態でブラウザの開発者ツールの「デバイスツールバー」(スマホ表示に切り替える機能)で確認するか、
実際にスマホで開いて、フォームの入力欄が見切れずに1列できちんと表示されるか確認してください。
問題なければ「スマホ表示のレイアウト修正」としてコミット。

## 【修正済み】中心星削除バグ (最優先だったバグ)


### 症状
最後の食べ物を削除したとき、中心星（および一部の星）だけ食べ物も星自体も消えない。

### 原因
`StarManager.js` の `addStar()` が、星の ID を「配列の長さ - 1」から決めていた。
星を1つ削除した後に新しい星を追加すると、ID が既存の星と重複し、
`starMap`（ID→星の対応表）内の既存の星の参照が新しい星に上書きされてしまっていた。
その結果、上書きされた星は `removeStarByInstance()` で二度と見つけられなくなり、
削除処理が静かに失敗していた（エラーも出ないため気づきにくいタイプのバグ）。

### 修正内容
- `src/universe/StarManager.js`
  - コンストラクタに `this._nextId = 0` を追加。
  - `generateId()` メソッドを新設し、常に増え続けるカウンターから ID を発行するように変更。
  - `addStar()` / `generateStars()` を `generateId()` を使うように修正。
- `src/app.js`
  - 保存データ読み込み時に、食事履歴 (`foods`) の各項目に `id` が無い場合は自動で補完する
    `migrateFoodEntries()` を追加（古いデータ形式でも削除ボタンが正しく効くようにするための保険）。

### 動作確認
実際に `npm run dev` で確認済み。中心星(A)の食べ物を1つずつ削除していき、
最後の1つも含めて正しく消えることを確認できた。
デバッグ用に一時的に入れていた `console.log` は削除済み。

問題なければ「中心星削除バグ修正」としてコミット。
